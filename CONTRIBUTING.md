# Contributing to SIGTERM

Welcome to the grid.

Before you contribute, read this. All of it. It's short.

---

## What SIGTERM Is

SIGTERM is a free, open source, browser-based multiplayer RPG inspired by BBS door games. It is MIT licensed. Anyone can run it. Anyone can contribute to it. Nobody owns the community except the community.

The mothership node at alpha.blipsandbloops.com is operated by pextris. That node is not a democracy. The code is open. The mothership is not.

---

## What Contribution Means

If you contribute code, artwork, lore, or anything else to this repository:

- You get credit. Always. Your name in the contributors list and in the Archive if your contribution is significant.
- You do not get ownership. Contributing code does not make you a partner, stakeholder, or decision maker on the mothership node.
- Your contribution is MIT licensed like the rest of the project. Anyone can use it, fork it, run it.
- The lore, characters, and story of SIGTERM belong to pextris. You can write lore contributions but the canon is maintained by the Sysop.

---

## What Running Your Own Node Means

If you run a community node using the SIGTERM installer:

- You are the Sysop of your node. Your node is yours.
- You can see your own player database. Nobody else's.
- You cannot see the mothership database. Ever.
- You cannot see other community node databases. Ever.
- Passwords never leave the node they were created on. This is a hard rule.
- The mothership shares aggregated information only — season data, leaderboard scores, Archive entries, selected Dead Drop messages. Never raw player data.
- You are responsible for your node and your community. The mothership is not liable for what happens on your node.

---

## The Hard Rules

These are not guidelines. These are rules.

1. Passwords never leave the node they were created on. Never.
2. Community node Sysops cannot access mothership player data. Never.
3. No node can access another node's raw database. Never.
4. GRT and credits have no real world value. They cannot be sold, traded, or exchanged for anything with real world value. Ever.
5. The game is free. It always will be. No feature that affects gameplay can be paywalled.
6. The network shares the story. Not the data.

---

## How to Contribute

- Bug reports — open an issue with steps to reproduce
- Feature requests — open an issue tagged enhancement, describe the problem you're solving not just the solution
- Pull requests — open an issue first and discuss before building. We don't want you to waste your time.
- Lore and writing — open an issue tagged lore. Canon decisions are made by the Sysop.
- Artwork — open an issue tagged art. The aesthetic is pixel art, cyberpunk, black backgrounds, neon.
- BBS sysops — if you want to run a node, use the installer. If you find problems with it, open an issue.

---

## What We're Looking For

- Bug fixes
- Performance improvements
- Accessibility improvements
- New OffOps content
- Lore contributions that fit the established universe
- ANSI artists
- BBS sysops who want to run nodes and report back

---

## What We're Not Looking For Right Now

- Major architecture changes without prior discussion
- New gameplay systems without prior discussion
- Monetization features of any kind
- Anything that creates a path from real money to in-game currency

---

## The Philosophy

The grid holds space. That's the whole design philosophy. Every decision — technical, social, economic — should be evaluated against that standard.

Does this hold space for runners?
Does this make the grid feel safer, warmer, more real?
Does this serve the community or extract from it?

If the answer to the last question is extract — it doesn't belong here.
